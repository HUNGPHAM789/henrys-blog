❌ The Problem

Building an EdTech platform solo is overwhelming. You need frontend, backend, AI integration, TTS, video processing, database design — and you need to ship fast before motivation dies. Traditional development: 1-2 features per week if you're lucky.

🔧 How It Works

Step 1: Boss describes the feature

"Merge all question types into one practice stage — video on top, 1 question at a time"

Step 2: Jarvis plans + executes

  • Reads the codebase to understand current architecture
  • Edits the Gemini prompt (API route) to restructure the output
  • Updates the lesson viewer (frontend) with new CSS + JS
  • Commits + pushes to GitHub in one atomic change

Step 3: Vercel auto-deploys

Push to master → Vercel builds → live in ~2 minutes. Boss refreshes and tests.

Step 4: Iterate in real-time

"The matching game looks dull" → Jarvis redesigns with gradient bg, shake animation, pill score badge → pushed in 3 minutes.
