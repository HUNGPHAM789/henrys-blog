The Productivity Automation Stack for Solo Builders

Calendar, email, and tasks — the three drains on every solo builder's day. Here's how Jarvis automates all three for Boss Henry.

Henry's Blog · March 24, 2026 · 9 min read

Solo builders are builders who also do their own sales, customer support, operations, content, and finances. You're the entire company. The only way to stay in flow is to ruthlessly eliminate admin overhead — and the tools available in 2025 make this more possible than ever before.

This post is specific to Boss Henry's situation: ESL teacher + app developer + blogger + AI experimenter. But the principles apply to any solo builder who uses an AI assistant. Let's go through the stack layer by layer.

Layer 1: Calendar Automation

The Problem

For an ESL teacher, the calendar is critical. Classes repeat on fixed schedules, but makeup sessions, rescheduling, and end-of-month reporting all require calendar-aware intelligence. Right now, Jarvis knows Boss Henry's timezone but doesn't have live calendar access baked into its default workflow.

The Solution: Google Calendar MCP + AI Scheduling

The Google Workspace MCP skill is already installed. With it active, Jarvis can:

  • Check today's schedule before starting any session (already logged in HEARTBEAT)
  • Auto-create reminder events when tasks are added to the active-tasks system
  • Pull class schedule data for the monthly ESL report without manual lookup
  • Block focus time automatically when Jarvis is running long tasks

Tools That Go Further

Reclaim AI (reclaim.ai) is the most intelligent calendar manager available. It learns your priorities and automatically schedules tasks, deep work blocks, and meeting prep. It integrates with Google Calendar and connects to Notion, Asana, and Todoist. The magic is that when your schedule changes, Reclaim reshuffles everything automatically — you never manually move calendar blocks again.

Cron/Notion Calendar (now built into Notion) is worth mentioning for Boss Henry specifically because Notion is already in the workflow. Events can be linked directly to project pages — your ESL class event links to the class notes, your Antera sprint event links to the sprint board.

Layer 2: Email Automation

The ESL Teaching Email Reality

ESL teachers deal with a specific email pattern: student questions, parent inquiries, class scheduling, payment reminders, and monthly reports to class groups. The volume isn't massive, but it's consistent and recurring — perfect for AI automation.

Gmail MCP + Jarvis = Zero Email Overhead

With the Google Workspace MCP active, Jarvis can:

  • Draft monthly class reports in Boss Henry's voice — sent to Jarvis as a task, reviewed, then sent
  • Auto-categorize incoming emails by student/class/priority
  • Flag any email that needs urgent response during the next Telegram check-in
  • Create calendar events from scheduling emails automatically

n8n — The Open-Source Automation Backbone

GitHub: github.com/n8n-io/n8n · Stars: 150,000+

n8n hit 150,000 GitHub stars in 2025, making it one of the most popular self-hosted automation tools ever. It's a visual workflow builder — connect any app to any other app through a drag-and-drop interface. No code required.

For Boss Henry, n8n could replace several manual workflows:

  • Zoom recording → Google Drive upload → teaching report generation → Telegram notification
  • New student email → Supabase record → welcome message → calendar event
  • Monthly invoice generation → payment tracking → follow-up reminders

n8n is self-hosted (free), has an AI agent mode (it can use LLMs to make decisions in workflows), and has 400+ app integrations. It's the automation infrastructure Jarvis could trigger rather than running all automation in the agent itself.

Activepieces — The No-Code n8n

GitHub: github.com/activepieces/activepieces · Stars: 12,000+

If n8n feels too technical, Activepieces is the more accessible alternative. Same concept (visual workflow builder, 450+ integrations) but designed for less technical users. It has built-in AI tools for adding LLM decision-making to workflows, and a managed cloud tier that requires zero DevOps.

Layer 3: Task Management

The Current System

Jarvis uses a combination of HEARTBEAT.md (reminders), memory/active-tasks.json (in-progress work), and Notion (project planning). This works but lacks a single task view that Boss Henry can consult without opening multiple tools.

Todoist + Jarvis Integration

API: developer.todoist.com · Jarvis integration: MCP via Zapier or direct API

Todoist's API is clean and well-documented. With a simple integration script, Jarvis could:

  • Create tasks in Todoist when new items are added to active-tasks.json
  • Mark tasks complete in Todoist when Jarvis finishes them
  • Read today's Todoist tasks as part of the morning briefing
  • Sync Todoist priorities with Jarvis's planning

The advantage of Todoist over Notion for tasks: it's faster to open, has excellent mobile apps, and has natural language date parsing ("add: finish Antera auth flow tomorrow 3pm").

The Bigger Picture: From Task Management to Life Management

The tools above handle the mechanics. But the real shift is philosophical: stop managing tasks yourself and start managing Jarvis who manages tasks for you.

This means:

  • Single input channel: Everything goes to Jarvis via Telegram. One place, always available.
  • Jarvis as task router: Incoming tasks go to the right system (Todoist for personal, Notion for project, active-tasks.json for in-progress Jarvis work)
  • Proactive surfacing: Jarvis doesn't wait to be asked — it pushes the right information at the right time via heartbeat checks
  • Automated reporting: Weekly summary of what was done, what's pending, what needs Boss's decision

The ESL teacher multiplier: The Zoom recording automation alone is worth 30 minutes per month. The email drafting for student communications is worth another 20. The calendar scheduling removes 2–3 weekly check-ins. Total: 3–4 hours/month of admin that just disappears. That's 40+ hours/year back for teaching, building, or rest.
