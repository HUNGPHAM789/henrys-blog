n8n: The Open-Source Automation Engine That Should Power Your AI Workflows

150,000 GitHub stars and growing. n8n is what Zapier wishes it was: self-hosted, code-capable, AI-native, and completely free.

Henry's Blog · March 24, 2026 · 8 min read

Zapier costs $20–100/month. Make is $9–16/month. And they're both cloud-only — your automation data flows through their servers, not yours. For most users, that's fine. For a solo builder with an AI assistant handling sensitive teaching data, student records, and business workflows, there's a better answer.

n8n (pronounced "n-eight-n") is the open-source automation platform that gives you Zapier's capabilities with none of the cost and all of the control. 150,000 GitHub stars — one of the fastest-growing developer tools of 2025. This is not a compromise choice: n8n is genuinely better for technical users than the paid alternatives.

What n8n Actually Does

n8n is a visual workflow builder. You create workflows by connecting "nodes" — each node is an action or trigger from a specific app. Nodes are connected by edges that define data flow and conditional logic. The result is a workflow that runs automatically when triggered.

Example: When a new student signs up on Antera (Supabase trigger) → create a Notion page for their notes → add them to a Google Calendar event → send a welcome message via Gmail → notify Jarvis via webhook → post to Telegram. That's 5 steps, 5 different apps, zero code.

Where n8n beats Zapier:

  • Self-hosted: Your data stays on your machine. No monthly bill, no rate limits you didn't set.
  • Code nodes: Any step can run arbitrary JavaScript or Python. When visual isn't enough, code.
  • AI agent mode: n8n has built-in LLM nodes — connect to Claude, Gemini, or any model to add AI decision-making to workflows.
  • 400+ integrations: Comparable to Zapier's library. Covers everything Boss Henry needs: Zoom, Google Workspace, Supabase, Telegram, GitHub, Notion, Slack.
  • Webhook support: Any workflow can be triggered by a webhook — meaning Jarvis can trigger n8n workflows with a single HTTP call.

The Jarvis + n8n Integration Pattern

Here's the key insight: n8n and Jarvis are not alternatives — they're complementary. Jarvis handles intelligence (reasoning, planning, writing, coding). n8n handles plumbing (moving data between services, triggering actions, scheduling recurring jobs).

The pattern:

  • Jarvis decides what needs to happen (e.g., "generate monthly teaching report")
  • Jarvis calls an n8n webhook to trigger the data collection workflow
  • n8n gathers data from Zoom API, counts sessions, pulls calendar events
  • n8n passes the structured data back to Jarvis via webhook response
  • Jarvis formats the report in natural language and sends it via Telegram

This separates concerns cleanly: n8n is the data plumber, Jarvis is the brain. Each does what it's best at.

Trigger: Supabase webhook on new user signup. Actions: Create Notion page → add to welcome email sequence → notify Boss → update student tracking spreadsheet.

3. Blog Post Pipeline

Trigger: Notion page status changes to "Ready to Publish." Actions: Export Notion content → convert to HTML → push to Git → deploy to GitHub Pages → Telegram notification → social media post draft.

4. Weekly Summary Report

Trigger: Every Friday at 6pm. Actions: Query Supabase for weekly Antera metrics → pull GitHub commit log → gather completed tasks from Jarvis → format weekly recap → send to Telegram.

5. Backup and Archive

Trigger: Daily at 2am. Actions: Export Supabase tables to JSON → upload to Google Drive → verify upload → clean up local temp files → notify on any failures.

Setting Up n8n on the Jarvis Machine

1

Install via npm

npm install -g n8n — runs on Node.js, already installed. Start with n8n — opens at localhost:5678.

2

Configure credentials

Add Google OAuth (already set up for other scripts), Zoom API credentials, Supabase API key, Telegram Bot token — all stored securely in n8n's credential vault.

3

Create the first workflow

Start with the weekly summary — it's the most general and immediately useful. Add a webhook trigger, pull data from each source, POST to Jarvis's notify endpoint.

4

Wire Jarvis ↔ n8n webhooks

n8n exposes webhooks; Jarvis can POST to them. Add a scripts/n8n-trigger.js helper that makes it trivial to trigger any n8n workflow by name.

5

Add to startup (optional)

Set n8n to run as a Windows service so it's always available. Use PM2 or the Windows Task Scheduler to start it on login.

The no-code angle: Once n8n is installed and credentials are configured, Boss Henry could build and modify workflows himself through the visual UI — no coding required. This is a meaningful upgrade to Boss's capability to automate things independently, without waiting for Jarvis to script everything.
