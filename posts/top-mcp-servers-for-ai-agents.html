Top 10 MCP Servers Every AI Agent Should Have

MCP turns your AI agent from a chatbot into an operator — able to search the web, query databases, write files, and trigger actions across your entire tool stack.

Henry's Blog · March 24, 2026 · 10 min read

Model Context Protocol (MCP) is one of the most important standards to come out of the AI agent world in 2024. Anthropic released it as an open standard, and within months it had become the universal plugin system for AI agents — every major framework (Claude Code, OpenCode, LibreChat, AnythingLLM) now supports it.

The idea is simple but powerful: instead of hardcoding tools into an AI model, you run lightweight servers that expose tools via a standardized API. Your AI agent connects to these servers and can call any tool — search the web, query Supabase, control a browser, send a Telegram message — without any custom integration code.

Jarvis runs on OpenClaw and already has several MCP servers configured. But there are gaps. This post covers the 10 most valuable MCP servers, which ones are already installed, and which ones Boss should add next.

What MCP Actually Does

Without MCP, every AI tool integration is a custom job. You write code to call the GitHub API, code to call the Supabase API, code to control a browser — and you maintain all of it separately. When you switch AI models, you rewrite everything.

With MCP, you run a server once and any MCP-compatible agent can use it. The agent sees a list of available tools: search_web, query_database, navigate_browser. It calls them like function calls. The server handles the actual API communication. You configure it once, use it forever.

For OpenClaw users, MCP is how Jarvis extends its capabilities without changing its core code. Every new MCP server is a new superpower for Jarvis.

The 10 MCP Servers That Matter Most

1. Supabase MCP

Lets your agent query, insert, update, and delete data in Supabase directly. For Jarvis: essential for managing Antera student data, ESL lesson content, and blog post records. Already installed with 20+ database tools. This is the highest-value MCP server for any serious project.

2. GitHub MCP

Reads repositories, creates issues, opens pull requests, searches code, and manages files directly from the agent. Instead of manually pushing to GitHub, Jarvis can open a PR automatically after building a feature. The official GitHub MCP server was released in 2025 and has become the standard.

3. Playwright / Browser-Tools MCP

Gives your agent a real browser. Navigate to URLs, click buttons, fill forms, take screenshots, extract text. The browser-tools MCP (which connects to a Chrome extension) is already installed on the Jarvis machine. For web scraping, testing, or automating any browser-based workflow.

4. Filesystem MCP

Controlled read/write access to local files. Your agent can read configuration files, write output, manage project directories. More secure than raw shell access — you define exactly which directories are accessible. Essential for any coding agent workflow.

5. Tavily Search MCP

Real-time web search optimized for AI agents. Unlike a basic search API, Tavily returns clean, structured results with full article content — not just links. Already available as a skill on the Jarvis machine. Use this instead of raw web scraping for research tasks.

6. Notion MCP

Read and write Notion pages, databases, and blocks. For Boss Henry: the Antera project notes, blog content, and ESL planning materials are all in Notion. Currently configured but using scripts/notion.js as a fallback due to MCP auth issues. Worth fixing — native Notion MCP would make content workflows much smoother.

7. Google Drive / Calendar MCP

Access Google Docs, Sheets, Calendar events, and Drive files. For the ESL teaching workflow: automatically read Zoom session data, update the teaching report Google Sheet, and check calendar for upcoming class schedules. The Google Workspace MCP skill is already on the Jarvis machine.

8. Zapier MCP

Access to 7,000+ apps through Zapier's automation platform. If something doesn't have its own MCP server, it probably has a Zapier integration. Think of this as the "long tail" MCP — Mailchimp, Slack, HubSpot, Typeform, Calendly. The trade-off: slower than native integrations and requires a Zapier subscription.

9. Memory MCP (mem0 or similar)

Persist and retrieve memories across sessions. As covered in the memory systems post, a memory MCP server would let Jarvis automatically store important facts from every conversation and retrieve them semantically. This is the biggest missing piece in the current OpenClaw setup.

10. Firecrawl MCP

Turn any website into clean, structured data for AI consumption. Firecrawl handles JavaScript-rendered pages, pagination, and anti-scraping measures that break simple fetchers. For research tasks, competitor analysis, or pulling content from sites that don't have APIs. 82,000+ GitHub stars — one of the fastest-growing tools in the space.

How to Install an MCP Server in OpenClaw

OpenClaw uses the mcporter CLI for MCP server management. The config lives at ~/.openclaw/workspace/config/mcporter.json. For Claude Code, the config is at ~/.claude/mcp.json.

The typical installation flow:

  1. Find the MCP server package (most are on npm or PyPI)
  2. Add it to your mcporter config: mcporter add <server-name>
  3. Configure authentication (API keys, OAuth tokens)
  4. Test with: mcporter call <server> <tool>
  5. Add to Claude Code config if you want Claude Code subagents to use it

Pro tip: The GitHub MCP server and Firecrawl MCP server are the two highest-priority additions for Jarvis right now. GitHub MCP would let Jarvis open PRs and manage the Antera repo without CLI wrangling. Firecrawl would supercharge research tasks like this blog series — no more wrestling with JavaScript-heavy pages.

The Security Question

Every MCP server you install is a potential attack surface. A malicious website could contain a prompt injection that tricks Jarvis into calling the Filesystem MCP to read sensitive files. A misconfigured database MCP could expose Supabase credentials.

Key rules:

  • Principle of least privilege: Filesystem MCP should only access specific directories, not your entire disk
  • Audit tool calls: Log every MCP tool call — who called it, with what arguments, and what it returned
  • Read-only by default: Start with read-only tools; add write permissions deliberately
  • Separate credentials: Each MCP server should have its own limited API key, not admin/root access
